# Product Context

## Why This Project Exists

This project exists to demonstrate fullstack development competency for a job interview. It showcases the ability to build a modern, production-ready web application using .NET for the backend and Angular for the frontend.

**Context:** Job application interview on Friday - this is a portfolio piece that must impress technical interviewers.

## Problems It Solves

1. **Interview Demonstration** - Provides a tangible example of fullstack development skills
2. **Technology Integration** - Shows proficiency in connecting .NET backend with Angular frontend
3. **Best Practices Showcase** - Demonstrates knowledge of modern development patterns
4. **Real-World Scenario** - Inventory management is a relatable business problem

## How It Should Work

### User Experience Goals

- **Professional & Polished** - Must look production-ready for interview
- **Fast & Responsive** - No lag or awkward loading states during demo
- **Intuitive Interface** - Easy to understand without explanation
- **Visual Feedback** - Clear indicators for all user actions

### Technical Workflow

1. **User Authentication** → Login with JWT token storage
2. **Dashboard View** → See inventory overview at a glance
3. **CRUD Operations** → Create, view, edit, delete inventory items
4. **Search & Filter** → Find items quickly
5. **Real-time Updates** → See changes reflected immediately

## Target Users

- **Primary:** Interview panel/technical evaluators
- **Secondary:** Potential future employers reviewing portfolio
- **Tertiary:** Personal learning and skill development

## Key Features (Interview Demo Focus)

### Core Demo Flow (Must Work Flawlessly)

1. **Login Screen** 
   - Clean, professional design
   - Demo credentials visible
   - Successful login transition

2. **Dashboard**
   - Inventory statistics cards
   - Low stock alerts (visual impact)
   - Recent activity

3. **Inventory Management**
   - List view with search/filter
   - Add new item (form validation)
   - Edit existing item

   - Success/error notifications

4. **Code Walkthrough Points**
   - Clean architecture explanation
   - API design decisions
   - Security implementation
   - State management approach

## Demo Script Talking Points

### Technical Highlights to Emphasize

- **Backend (.NET):**
  - RESTful API design
  - Entity Framework Core with PostgreSQL
  - JWT authentication & authorization
  - Repository pattern for data access
  - Input validation and error handling
  - Swagger documentation

- **Frontend (Angular):**
  - Component-based architecture
  - Reactive forms with validation
  - HTTP interceptors for auth tokens
  - Route guards for protected pages
  - RxJS for state management
  - Responsive design

- **Full-Stack Integration:**
  - CORS configuration
  - Environment-based configuration
  - Error handling across layers
  - Type safety (C# + TypeScript)

## Interview Presentation Strategy

### 5-Minute Demo Flow

1. **Introduction (30 sec)**
   - Project overview
   - Tech stack mention

2. **Live Demo (3 min)**
   - Login demonstration
   - Show dashboard
   - Create an item
   - Edit an item
   - Search/filter
   - Delete an item

3. **Code Walkthrough (1 min)**
   - Show clean architecture
   - Highlight one interesting implementation detail

4. **Q&A Preparation (30 sec)**
   - Be ready to discuss:
     - Why .NET + Angular?
     - Security considerations
     - Scalability thoughts
     - What you'd improve with more time
